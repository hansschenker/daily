It is not either React or Angular!
React: Is a Library using Javascript with a bit syntactic sugar (tsx) on it. 
React uses Javascript with good pattern practices coding style. In the beginning you are faster with React, 
but when the Project gets bigger, I think Angular with its standard Project structure and the help of the Angular CLI is easier to manage 
In React you can learn good Javascript coding style with a component oriented architecture..
Angular: is a Framework that puts Framework Directives into Html and the Angular Compiler has then to compile this Html filled with Directives into valid Html and Javascript. Furthermore Angular uses Typescript as Language and this gives strong typing support in the code which is necessary to refactor your code without fearing that you break the code. Typescript is a huge step forwardin Javascript Land. There is a saying, that big Javascript Projects are readonly projects! Readonly because they are so  big and you lost the overview, that you are afraid to change the code. I think in a few years time React will also use Typescript. Angular with the need to compile the Html is further away from the Web Platform than React but with the strong Typing from Typescript has a huge advantage over React in analysing the code and finding ways to improve performance.
React is very good for prototyping components because of its lightness and no big ceremony upfront.You can learn good coding style in React.
Angular is very good for enterprise size projects. It is heavy to begin with, but once you have setup you can add many features without loosing
managability of the project.
React and Angular and also Vue are here to stay for a longer time on the web.
